- data/ - Pacote centralizado para todo o tipo de dados da aplicação 
    - src/
        - attachments/
            - gateway.ts - Mantém a integridade referencial dos blobs. Evita apagar arquivos em uso, sincroniza o db
                * Ordem: primeiro o arquivo é sempre salvo no disco e o hash é armazenado no db
                * refCount: Mostra em quantos locais o arquivo faz referencia, evitando apagar ou manter arquivos inúteis
                * gc: busca arquivos sem referências e apaga
            - storage.ts - Gestão de armazenado de blob utilizando `SHA-256` como key
                * nome das pastas: Utilizar 2 itens como prefixo salvar arquivos em até `256` pastas, evitando lentidão conhecida no FAT32/NTFS/ext4 (Windows e Linux)
                * O uso da chame `SHA` evita que arquivos iguais sejam duplicados no disco
        ----------
        - backup/ - Portabilidade e Backup (Faz o backup da história)
            - export.ts - Transforma toda a história em um .zip
            ----------
            - import.ts - Restauração da história
            ----------
            - manifest.ts - Define o contrato de validade, informa exatamente o que há dentro do .zip
        ----------
        - events/
            - event-store.ts - É a fonte de verdade absoluta da sessão
                * Utiliza JSONL para que cada linha seja um objeto independente
                * append: filtra qualquer tipo de "lixo"
                * read: faz a leitura de arquivos em pequenos pedaços, evitando eventuais travas
                * Open-Write-Close: sempre fecha arquivos para evitar o limite de open files
                * readAfter: auxiliar completo de `replay.ts`
            ----------
            - reducer.ts - Recebo um evento e decide como essa informação deve alterar o db
            ----------
            - replay.ts
                * rebuildProjection - Método utilizado quando precisamos mudar a estrutura do db ou sincronizar 
                * catchUp - "Continuar de onde parou", evitar processar tudo novamente toda vez que o app abre
                * Idempotência - mesmo que seja executado 1000x o resultado sempre será o mesmo 
                * Resiliência - Se houver algum tipo de erro, o JSONL continua salvo no disco
                * Auditoria - Acesso aos metadados completos
        ----------
        - migrations/
            - backup.ts - Realiza um backup do banco de dados antes de aplicar uma migração
            ----------
            - runner.ts - Executa as migrações dentro do banco de dados
        ----------
        - schema/ - Possui vários arquivos com a listagem estrutural da tabelas do banco
        ----------
        - sqlite/
            - database.ts - Gestão de conexão com o db e métodos disponíveis
                * open - Localiza ou cria o arquivo .db no disco
                * prepare - Compila uma query antes de executar (Proteção para SQLi)
                * exec - Executa ações sem esperar um retorno. Ideal para DDL
                * pragma - "botões de ajuste"
                * transaction - Garante que 100% da transação foi realizada ou faz o rollback
                * close - Fechar o arquivo do db de forma segura
                * dispose - Descarta o processo
            - error.ts - Gestão de erros da conexão SQLite, apenas open/closed
        ----------
        - drizzle.ts - Arquivo de configuração central do drizzle
        ----------
