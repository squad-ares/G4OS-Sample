- sources/ - Pacote centralizado de integrações externas
    - src/
        - interface/
            - registry.ts
            ----------
            - source.ts
        - lifecycle/
            - intent-detector.ts - Uma análise de intenções do usuário para definir quais fontes precisa fornecer para o modelo, evitando enviar todos os itens de uma vez (Talvez tenha formas mais precisas de implementar isso)
                * Faz uma estratégia de cascata seguindo uma ordem de prioridade
            ----------
            - lifecycle-manager.ts - Gerenciamento de contexto, follow-up de perguntas
                * sticky: Fixa as integrações utilizadas na sessão. Assim é possível recuperar o contexto de tudo que o usuário pediu
                * rejected: Armazena o que falou ou foi removida. Auxiliando para que a ferramenta não seja reativada a cada chamada
                * planTurn: Cruza as informações salvas e rejeitadas e divide em `nativeSources` (coisas que o modelos ja sabe fazer) e `brokeredSources` (Integrações de terceiros)
                * activateBrokered: Tentativa de ligar cada integração
        ----------
        - managed/
            - base.ts - É uma classe base para implementar conectores (Slack, Drive e etc)
                * BehaviorSubject: Garante que a tela não fica presa carregando, e recebe a resposta de eventos
        ----------
        - mpc-http/
            - reconnect.ts - Monitora a request HTTP e tenta refazer reconexão em caso de quedas
                * skip(1): Garante que a reconexão não sera feita no momento da primeira instancia da conexão
            - factory.ts - Responsável pela execução real dos processos via MCP
            - source.ts - Faz a gestão remota com servidores externos HTTP que utilizam o protocolo MCP
        ----------
        - mcp-stdio/
            - factory.ts
            - runtime-mode.ts - Define onde o processo será executado, direto na máquina, container e etc
            - source.ts - Faz a gestão da conexões locais
                * stdio: Standard Input/Output
        ----------
        - oauth/ - Regra RFC 8252
            - flow - faz toda a gestão do processo de oauth
                * PKCE: Senha descartável para cada tentativa de login
                * Ataque CSRF: Impede que um site malicioso force um login forjado
            ----------
            - callback-handler.ts - Gerencia o fluxo de callback do OAuth
            ----------
            - loopback - Utiliza um servidor local na ausência do DeepLink
                * É uma forma de ter redundância na autenticação
            ----------
            - pkce - Geração da chave temporária
                * Impermeável por brute force
