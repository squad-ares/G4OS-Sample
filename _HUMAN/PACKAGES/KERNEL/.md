- kernel/ - Pacote centralizado de dependências que todo o projeto deve consumir. Aqui tratamos todo tipo de validação, erros e utilitários
    - src/
        - disposable/ - EVITA vazamento de memoria. "Coração da limpeza" - Inspirado no VSCode
            - disposable-base.ts - (CORPO) É um facilitador de uso para o `disposable-store`
            - disposable-store.ts - (MOCHILA) Funciona como um balde, ao invés de precisar ficar lembrando de descartar vários recursos, ele centralizar e gerencia isso.
                * add - Adiciona um item na mochila
                * delete - Remove um item da mochila, mas sem ter interrompido seu processo (deletar != descartar)
                * deleteAndDispose - Remove um item da mochila e descarta imediatamente
                * dispose - Descarta todos os itens da mochila
            - helper.ts - São auxiliares para facilitar alguns processos de descarte. Eles não estão completamente relacionados com o `disposable-store` nem com o `disposable-base`
            * toDisposable - Converte uma função comum em um disposable
            * combinedDisposable - Agrupa vários disposables em um único
            * bindToAbort - Descarta automaticamente se o AbortSignal for abortado
            - types.ts
                * Fornece uma estrutura minima, a única opção é que ele tenha um botão "dispose" (descartar)
        ----------
        - erros/ - Existem vários arquivos nessa pasta com o mesmo intuito. Mapear os possíveis erros do projeto. Com isso nos nunca precisaremos lançar um erro manualmente. Eles sempre utilizam `ErrorCode` para fazer um mapeamento mais fácil.
        ----------
        - logger/ (Ainda precisamos testar algumas coisas para entender o fluxo completo dos recursos)
            - logger.ts - Maneira centralizada de gerencias nosso logs. Com camadas de segurança
                * REDACT_PATHS - Caminhos de segurança que não devem ser expostos nos logs

        ----------
        - schemas/ - Seguindo o mesmo exemplo do erros, aqui temos uma mapa completo de validação dos dados em runtime. Evitando que os estados sejam criados de forma inválida ou incompleta.
        ----------
        - serialization/
            - json.ts
                * serializeJson: COnverte JSON para string
                * deserializeJson: Converte JSON para string validando a informação contra um schema (ZOD)
                * serializeJsonl: COnverte JSON para string inline
                * deserializeJsonl: Converte JSON para string validando a informação contra um schema (ZOD) em várias linhas
        ----------
        - types/ - Aqui convertemos as definições do ZOD para typescript. Garantindo que os objetos passem por uma pré-validação antes de serem usados.
        ----------
        - validation/
            - builders.ts - Possui estruturas ZOD que são muitos comuns e utilizadas em várias partes do projeto
            - env.ts - Verifica se as variáveis de ambiente foram configuradas corretamente
            - parse.ts - Inspeciona e formata eventuais erros de validação
