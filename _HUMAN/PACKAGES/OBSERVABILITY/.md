- observability/ - Pacote centralizado para monitoramento e coleta de métricas da aplicação
    - src/
        - debug/
            - export.ts - Permite que o usuário exporte os logs manualmente
                * Passa por funções de segurança para remover dados sensíveis
            ----------
            - redact.ts - Realiza a limpeza dos textos e objetos (Focado no desenvolvedor)
        ----------
        - memory/
            - leak-detector.ts - Investiga "Quem" está roubando memória
                * Focado em Event Listeners órfãos
            ----------
            - memory-monitor.ts - Monitoramento proativo
                * Dispara alarmes se um processo  forçar mais que o esperado
                * Monitora quando a memória está crescendo rápido demais
                * checkThresholds: Valida o limite absoluto (RSS), valida o crescimento relativo (Heap Growth)
        ----------
        - metrics/
            - registry.ts - Centraliza todos o "sensores" que serão monitorados no sistema (Reporta números)
            ----------
            - timer.ts - Mede a latência agregada
        ----------
        - sdk/
            - init.ts - Gerencia a instalação do OTL e realiza o envio de dados para o servidor
        ----------
        - sentry/
            - init.ts - Reconstrução do DOM, coleta de erros pelo sentry
            ----------
            - scrub.ts - Filtro de privacidade
        ----------
        - propagation.ts - Garante que o tracer não se perca na mudança de um processo para o outro
            * Main Process para um Session Worker
            * injectTraceContext: Identifica de onde o processo está sendo executado
            * runWithExtractedContext: É a passagem da informação
        ----------
        - tracer.ts - Realizado o monitoramento de "Quanto tempo levou" e "Onde"
            * Cada etapa de um processo é um span. Ex: clica no botão -> Validar login (trace1) -> Gravar no JSONl (trace2)
        ----------
