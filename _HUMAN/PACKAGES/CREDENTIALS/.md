- credentials/ - Pacote centralizado para gestão de credenciais do usuário
    - src/
        - backends/
            - file-backend.ts - É quem executa as ações dentro do processo
                * BASE64URL: Remove caracteres base de arquivos, garantindo a compatibilidade entre plataformas
            ----------
            - in-memory-backend.ts - Utilizado em test, sem criptografia e sem persistência
            ----------
            - safe-storage-codec.ts - É a ponte completa entre o APP e a OS
                * É o local onde há comunicação segura para coletar as chaves
                * MacOS: Keychain Access
                * Windows: DPAPI (Data Protection API)
                * Linux: Secret Service ou KWallet
                * ATENçÃO: Se alguém roubar o arquivo de segurança, e levar para outro disco ele não conseguira ler as chaves
        ----------
        - migration/
            - migrator.ts - Migra as chaves da V1 para V2 adotando os novos critérios de segurança
            ----------
            - v1-reader.ts - Faz as leituras das chaves da V1
        ----------
        - rotation/ - Tratamento para atualizar as credenciais periodicamente.
            - handler.ts - "Equipe de renovação", define quando a "Bateria" precisa ser trocada
                * Aqui temos apenas a aplicação da interface.
            ----------
            - oauth-handler.ts - É um dos tipos que implementa rotação.
                * Resolve problemas de tokens que expiram rápido
                * O processo de hidratação é bem similar ao da WEB, utilizando refreshToken
                * Se tentarmos rotacionar uma chave `oauth.google_drive`, o orquestrador saberá que esse é o handler que precisa ser usado
                * Executado a cada 5 minutos
                * Fluxo: Pega no vault, Verifique as que possuem expiresAt e tem 5 minutos para vencer, busca o handler correto, executa isoladamente, sem interromper o fluxo de varredura.
                * OBS: Permite que tarefas e agentes sejam executados por dias, sem sofrer interrupção.
            ----------
            - orchestrator.ts - Garante que o sistema nunca pare por causa de uma chave expirada.
                * Buffer: O script não espera a chave expirar, o que causaria interrupção do serviço
                * Padrão de 5 minutos, se a chave expira 12:00, tentamos renovar 11:55
        ----------
        - factory.ts - Decide qual moto utilizar, para evitar criptografias em desenvolvimento, facilitando o processo
            * InMemoryKeychain: Efêmero, some quando o teste termina
            * FileKeychain + Plaintext: Salva na pasta secrets mas em texto puro
            * FileKeychain + SafeStorage: Salva no disco, dados ilegíveis sem chave mestre do SO
            * loadSafeStorageCodec: Carrega de forma assíncrona pois, em prod são pacotes pesados
            * CODEC: Coder Decoder, é a camada de transformação entre a OS e o APP

        ----------
        - vault.ts - Arquivo que lida com todas as ações das credenciais
            * Lida com problemas de concorrência e corrupção
            * set: Escreve a variável
            * delete: Apaga a variável
            * list: Retorna todas as variáveis (Sem valor)
            * exists: Verifica se a variável existe
            
        ----------
