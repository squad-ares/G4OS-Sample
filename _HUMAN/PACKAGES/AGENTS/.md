- agents/ - Pacote centralizado para controlar todas as implementações de agentes de maneira agnóstica ao uso
    - src/
        - interface/
            - agent.ts - Define todas as configurações e eventos que um agente pode ter e receber.
            ----------
            - registry.ts - Registra a utilização do agente e disponibiliza para a utilização (desktop)
            ----------
            - schema.ts - Valida todo tipo de implementação e configuração passada, evitando eventuais quebras no processo.
        ----------
        - claude/ - Estrutura completa para execução via Claude
            - config/
                - mapper.ts - Traduz as configurações literais do APP para valores reis implementados juntos da API
            ----------
            - prompt-cache/
                - cache-markers.ts - Realiza os marcadores para economizar o consumo repetitivo de tokens já processados
                    * applyPromptCache: coloca as marcações (ephemeral)
            ----------
            - providers/ - Aqui estão as configuração de "Quem" pode fornecer acesso aos modelos da Anthropic
                * Existe mais formas de ter acesso aos modelos Claude, além da API oficial, eles são configurados aqui
            ----------
            - runner/
                - event-mapper.ts
                ----------
                - stream-runner.ts - Faz a gestão dos eventos de stream
                ----------
                - tool-accumulator.ts - Trabalha montando os chunks para compor uma resposta final
            ----------
            - capabilities.ts - Define as capacidades do modelo anthropic, fornecidas ao usuário
            ----------
            - claude-agent.ts - Realiza a implementação concreta do agente
            ----------
            - factory.ts - Define qual o provedor fornece acesso ao Claude
                * Atualmente, com uso da AWS, acredito que outros providers do claude deixarão de ser executados
            ----------
            - index.ts - Arquivo central de exportação do agente
            ----------
            - types.ts - Tipagens específicas do agente claude
        ----------
        - codex/ - Foi projeto para rodar um motor de IA localmente, dentro do computador do usuário.
            * Utiliza comunicação primitiva com o programa inciado, stdin/stout
            - app-server/
                - client.ts -  É o responsável pelo processo de comunicação entre os dois sistemas
                ----------
                - event-mapper.ts - Traduz os eventos do subprocesso para o formato reconhecido pela aplicação
                ----------
                - frame.ts - Engenharia defensiva, para evitar que o conteúdo chegue quebrado ou mal-formatado
                ----------
                - input-mapper.ts - Traduz para o formato aceito dentro da aplicação
                ----------
                - node-spawner.ts - Responsável pela inicialização do subprocesso (Como se desse um duplo clique no exe)
                ----------
                - protocol.ts - Definição das regras, define a como a comunicação deve acontecer e o que ela aceita
                ----------
                - subprocess.ts - Interface de utilização do subprocesso
                ----------
            - bridge-mcp
                - connect.ts - Permite a conexão com tools nativas da computador
            ----------
            - binary-resolver.ts - Procura o local de instalação do binário
            ----------
            - codex-agent.ts - Implementação concreta do agente
            ----------
            - factory.ts
        ----------
        - google/
        ----------
        - openai/
        ----------
        - permissions/
            - default-resolver.ts - Atua como um Juiz da sessão, verificando as aprovações e recusa do usuário.
                * Aqui é onde o `permission-handler` e `safe-allowlist` fazem sentido.
            ----------
            - queue.ts - Responsável pela tela "congelar", enquanto o usuário não clica em um botão
                * Previne 'zumbis', se a sessão é encerrada, as promessas são fechadas com o contexto de `queue_disposed`
            ----------
            - safe-allowlist.ts - Regas rígidas que não podem ser burladas
                * SAFE_MODE_ALLOWED_TOOLS:  Todas ferramentas read-only
                * SAFE_MODE_FORBIDDEN_TOOLS: Comandos que não podem ser executados, eles são cortados pelo Kernel antes de serem processados.
                * classifyForSafeMode: Se não conhecer assume `unknown` e aplica a regra padrão
        ----------
        - shared/
            - broker/
                - mcp-pool.ts - Define o contrato que um MCP dve seguir para ser implementado
                    * Suporta stdio (local) e http (remoto)
                ----------
                - permission-handler.ts - Responsável pelo HITL 
                    * UAC: User Account Control
                    * Verifica a natureza dos eventos e gerencia as permissões de execução
                ----------
                - session-tools.ts - Gerencia as ferramentas necessárias por sessão
                    * Desabilitada para o Gemini, pois já existe recurso nativo
                    * Ordenação de prioridade + ordem alfabética, para manter a economia de token ja processados
                ----------
                - source-activation.ts - Gerencia as conexões/autenticações das fontes
                    * detectBrokeredSourceActivation: Detecta ativação por parte do usuário e retoma o fluxo
                    * not_enabled: Instalada mas desligada
                    * auth_required: Ligada mas o token expirou ou é o primeiro uso
                    * incompatible: Conflito de versão
                    * ok: Tudo certo

            ----------
            - thinking/
                - level-resolver.ts - Define como o modelo de pensamento deve se comportar em diferentes provedores de AI
        ----------
        - streaming/
            - backpressure.ts: Focado na sobrevivência do processo, definindo eventos importantes e pausas para continuar executando processos.
                * É acionado somente quando necessário, impedindo que a aplicação trave completamente.
            - batch-deltas.ts: Cadencia a as respostas em 16ms o que no frontend representa 60FPS. Isso garante uma taxa de att constante sem bater 100% do processamento, principalmente em modelos que respondem muito rápido.
                * Valida se o texto é do tipo delta para não gerar atraso em comando vitais como tool_use por exemplo.
