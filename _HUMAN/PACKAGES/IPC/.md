- ipc/ - Pacote centralizado para todos os contratos IPC, evitando possuir vários handles em um único tRCP
    - src/
        - server/
            - middleware/
                authed.ts - Camada de autenticação para usuários logados
                logging.ts - Registra o traceId, registra o início/fim de requests e registra os logs
                rate-limit.ts - Limite de taxa para usuários
                telemetry.ts - Registra o spanId, registra o início/fim de requests e registra os logs
            ----------
            - routers/ - Essa pasta possui vários contratos utilizando o ZOD como informação de "O que aceitar", além de tratamento de erros.
            ----------
            - context.ts - É a "caixa de ferramentas" que cada request possui. Aqui possuímos apenas a interface de cada requisição, não implementamos ou executamos nada. também abstraímos o conceito do Electron para não depender de ele.
            ----------
            - root-router.ts - O agrupamento de todos os contratos definidos em `routers`
            ----------
            - trpc-base.ts - Aqui aplicamos dois conceitos importantes. O transformer, onde utilizamos o superjson para que as informações de erro não "derretam". Formatação de erro, realizamos a limpeza do erro e passamos em um formato que a renderização consiga entender.
            ----------
            - trpc.ts - É o pipeline padrão, todas as chamadas de usuários já passam por logging e telemetry, sem precisar criar "console.log" em cada procedure.
        ----------
        - shared/
            * superjson-setup.ts - Registra todas as classes de superjson. O JSON.stringify é muito limitado, ele so consegue entender tipos primitivos de dados. Na passagem de Main para Renderer, ele vira um JSON comum
