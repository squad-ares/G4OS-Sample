- platform/ - Pacote centralizado onde tratamos todas a divergências necessárias entre plataformas
    - src/
        - keychain/
            - interface.ts: Define a forma como sistema deve tratar informações sensíveis de forma segura. Keychain interage com ferramentas de segurança nativas do computador. Os dado são criptografados a nível de disco e so podem ser acessados por usuários logados.
                * set: Salvar a chave
                * get: Coletar a chave
                * delete: Remover a chave
                * list: Retorna os nomes das chaves (sem valores). Útil para mostrar quais as chaves conectas sem expor valores.
        ----------
        - spawn/
            - interface.ts: Implementa critérios de segurança e governança para qualquer processo, evitando a utilização direta do `child_process.spawn`.
                * timeoutMs: Previne que um processo trave e rode para sempre. O processo "morre" no tempo determinado se não for finalizado.
                * killSignal: Define como o processo deve ser finalizado. SIGTERM: Amigável, SIGKILL: Destrutivo/Forçado.
                * autoKillOnExit: Define que se o o aplicativo principal fechar, os processos morrem. Evitando processos zumbis. (Crítico para evitar consumo de CPU em segundo plano.).
                * memoryLimitMb Define o limite de memória que um processo pode consumir. Sem isso, um processo com erro pode consumir toda a memória do sistema.
        ----------
        - paths.ts 
            - Define os principais caminhos úteis da aplicação
                * getAppPaths: Retorna todos os `paths` 'úteis, evitando acessar de diferentes formar em diferentes lugares, fazendo um switch automático baseado no tipo de distribuição
        ----------
        - platform-info.ts
            - Agrupa as informações estruturais da plataforma. Evitando o uso de `process.platform` em outros lugares.
                * detectOsFamily: Detecta e faz um parse amigável para o tipo da plataforma do usuário (Não exportada)
                * detectArch: Detecta qual arquitetura do usuários, limitado atualmente apenas para x64 e arm64 (Não exportada)
                * detectWsl: Verifica se o sistema esta sendo executado dentro de um WSL (Não exportada)
                * getPlatformInfo: Função principal que retorno todo o contexto de hardware previsto em `PlatformInfo`
        ----------
        - runtime-paths.ts
            - Arquivo que utilizamos para centralizar todos o caminhos da aplicação que são definidas em runtime
                * initRuntimePaths: É chamada no start da aplicação definindo o caminho base onde devemos buscar todas as demais execuções. Isso evita uma definição manual para cada item.
                * requireLocation: Apenas uma função auxiliar para controle de erro, caso o usuário tente utilizar o path sem o valor definido pela `initRuntimePaths`
                * runtime: Um objeto com definição de todos o runtimes do projeto. Sempre que um novo runtime for necessário, deve ser inserido aqui. NUNCA solto no código.
                * validateRuntimeIntegrity: Uma função auxilia rde checagem dos itens de runtime. Atualmente valida os itens mais criticos.
                * isMacOS, isWindows, isLinux: Auxiliares rápidos booleans
        ----------
